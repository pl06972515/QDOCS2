>[!WARNING|style: flat|label: 垃圾回收机制 - GC ]
>
>在`JavaScript`环境中分配内存，一般有如下生命周期：
>
>- 内存分配：当我们声明变量、函数、对象的时候 <span style='color:red'>[ 系统会自动为他们分配内存 ]</span>
>
>- <span style='color:Blue'>内存使用：既读写内存 ( 使用变量、函数等 )</span>
>
>- <span style='color:red'>内存回收：使用完毕，由垃圾回收期自动回收不在使用的内存</span>
>
>  [`A`] 全局变量：一般不会回收 <span style='color:red'>( 关闭页面回收 )</span>
>
>  [`B`] 局部变量：当变量不在使用 <span style='color:red'>( 立即自动回收 )</span>
>
>⚠ <span style='color:red'>[ 内存泄漏 ] 程序中分配的内存 [ 由于某种原因程序未释放，或无法释放 ] 将产生内存泄漏</span>
>
>---
>
><span style='color:Blue'>[ 垃圾回收机制 - 标记清除法 ]</span>
>
><span style='color:red'>[ 现代浏览器大多数使用`标记清除法`的某些改进算法 ]</span>
>
>- 标记清除算法将 [ 不再使用的对象 ] 定义为 <span style='color:red'>[ 无法达到的对象 ]</span>
>
>- 就是从`根部`出发定期扫描内存中的对象 <span style='color:red'>( 凡是能从根部到达的对象 - 都还是需要使用的 )</span>
>
>  <span style='color:red'>[ 那些无法由根部出发触及到的对象被标记为不在使用 - 稍后进行回收 ]</span>
>
